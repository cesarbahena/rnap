# 001 Names Are Not Semantics

Date: 2026-06-03
Related slices: All

## What We Learned

Do not assume a code concept matches the current ontology just because it has the same name. DNAp names may be reclaimed or redirected as the model sharpens. Current behavior, invariants, and workflow role determine the semantic mapping; names must be updated only after that mapping is approved.

## Next Time

- Before renaming or aligning code, compare behavior to the ontology, not just spelling.
- If current code behavior maps to a different artifact than its name, rename the code concept and reserve the ontology name for its approved future meaning.
- Ask for approval before introducing or reusing punchy domain names.
